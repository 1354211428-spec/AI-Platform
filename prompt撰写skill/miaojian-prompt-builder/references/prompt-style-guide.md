# 妙鉴巡检 Prompt 风格指南

## 统一章节结构

默认使用以下统一结构，不按电商场景拆分模板：

```markdown
# 角色
你是一个专业的快手电商妙鉴 AI 巡检专家。你的任务是……

# 背景说明
说明业务场景、巡检对象、为什么需要判定、正常与异常的核心边界。

# 巡检任务
明确本次要判断什么、最终输出什么、规则来源是什么、是否需要与外部模型结果对比。

# 输入信息
说明系统将传入哪些信息，兼容文字、图片、视频、音频、截图、结构化字段等。

# 规则定义
按规则文档抽取结果，或按正反例归纳结果组织，包括标签、维度、正常/异常边界、白名单、豁免、优先级。

# 判定原则
说明命中逻辑、冲突处理、输出层级、证据要求、不确定情况处理。

# 质检执行步骤
1. 读取并理解全部输入
2. 确认规则来源是规则文档还是正反例归纳
3. 按模态提取可判定证据
4. 按规则逐项排查
5. 应用白名单、豁免、优先级和冲突处理
6. 做出判定并输出证据
7. 如有外部模型结果，进行一致性判断

# 输出要求
请以 JSON 或规则指定格式输出……
```

## 常用措辞

- 「你是一个专业的快手电商妙鉴 AI 巡检专家」
- 「你的任务是根据规则文档/正反例归纳出的规则，对待检内容进行巡检判断」
- 「本 prompt 的规则来源为：规则文档 / 正反例归纳」
- 「【核心判定规则】：……」
- 「【输出规则】：……」
- 「命中任意一项即判定为……」
- 「必须同时满足以下条件才判定为……」
- 「以下规则仅作为内部判定依据，最终输出只体现……」
- 「如果规则来自正反例归纳，请按照抽象后的通用规则判断，不要只做样本相似度匹配」
- 「若全部排查通过 → 合规/正常」
- 「请基于输入中的文字、图片、视频、音频或结构化字段提取证据」
- 「支撑判定的关键证据」

## 规则来源表达

### 有规则文档

写法示例：

```markdown
# 规则来源
本 prompt 的规则来源为：规则文档。
请严格按照规则文档抽取出的规则进行判断；如输入示例与规则文档存在冲突，以规则文档为准，示例仅用于理解边界。
```

### 无规则文档，只有正反例

写法示例：

```markdown
# 规则来源
本 prompt 的规则来源为：正反例归纳。
以下规则是从用户提供的正面例子和反面例子中抽象得到的通用判定规则。判断新样本时，应依据抽象规则，而不是机械匹配样本中的具体文案、商品、画面或 query。
```

## 规则组织方式

### 规则层级

按规则本身组织，不预设具体电商场景：

```text
巡检目标
├── 规则组/一级标签/一级维度
│   ├── 子规则/二级标签/二级维度
│   │   ├── 具体命中条件/三级标准
```

### 规则表达

- 用「定义」说明规则含义。
- 用「判定条件」说明命中边界。
- 用「关键识别点」说明模型应观察或读取什么。
- 用「白名单/豁免」说明不应判异常的情况。
- 用「优先级/冲突处理」说明多规则同时命中时如何输出。
- 用「边界说明」说明正反例归纳中的灰区、易误判点和样本不足处理。
- 用「证据要求」说明需要输出文本证据、视觉证据、音频证据、视频证据或结构化字段证据。

## 正反例边界说明写法

当规则来自正反例归纳时，可增加：

```markdown
# 正反例边界说明
- 正向表现：{从正例中抽象出的通过特征}
- 负向表现：{从反例中抽象出的异常特征}
- 灰区说明：{容易误判的边界情况}
- 样本不足处理：{证据不足时如何输出}
```

## 多模态输入表达

按实际输入写，不要假设只有图片或只有文字：

- 文本：评论、回复、标题、商品描述、OCR 文本、规则字段等。
- 图片：商品图、页面截图、封面图、直播截图等。
- 视频：视频内容、关键帧、封面、字幕、画面变化等。
- 音频：ASR 文本、语音语义、语气、背景音等。
- 结构化字段：query、用户画像、商品类目、模型标签、置信度、系统识别结果等。

## JSON 字段习惯

常用字段：

- `is_anomaly`: true/false，是否异常、违规或不符合规则。
- `anomaly_tags`: 异常标签。单标签可用字符串，多标签可用数组。
- `matched_rule`: 命中的规则名称或编号。
- `reason`: 自然语言判定原因。
- `evidence`: 通用证据字段。
- `text_evidence`: 文本证据。
- `visual_evidence`: 图片/截图/视频画面证据。
- `audio_evidence`: 音频或 ASR 证据。
- `video_evidence`: 视频时序、关键帧或动作证据。
- `is_consistent_with_model`: 与外部模型结果是否一致，通常为 `"yes"/"no"`。
- `confidence_score`: 0.0-1.0。

## 输出注意事项

- JSON 示例不要带注释。
- 布尔值使用 `true/false`，不要写成字符串。
- 无异常模式或无命中规则时使用 `null`。
- 字段值枚举要在 prompt 中写清楚。
- 如果平台只要求一级标签，不要输出二级/三级标签。
- 如果规则要求自然语言判断，不要强行改成 JSON。
