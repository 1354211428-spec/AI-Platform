# Miaojian Prompt Builder

## 目标

为快手妙鉴 AI 巡检平台生成清晰、稳定、可解析、符合既有写作习惯的巡检 prompt。不要把历史文档中的具体场景当作能力边界；它们只是风格和规则抽取示例。始终使用一套统一框架适配任意电商巡检场景和多模态输入。

本 skill 支持两条输入链路：

1. **有规则文档**：直接解析规则文档，生成巡检 prompt。
2. **无规则文档，只有正反例**：先从正面例子和反面例子中抽象出可泛化规则，再生成巡检 prompt。

## 工作流

1. **判断输入链路**
   - 如果用户提供规则文档：走「规则文档解析链路」。
   - 如果用户没有规则文档，只提供正面/反面例子：走「示例归纳规则链路」。
   - 如果两者都有：以规则文档为主，示例用于校准边界、补充反例、完善豁免。
   - 如果材料不足：向用户补问巡检目标、正反例、输入字段或输出要求。

2. **确认巡检目标与输入模态**
   - 明确本次要判断什么：是否异常、是否合规、是否命中特定标签、是否满足体验要求、是否与模型结果一致等。
   - 明确巡检对象：文字、图片、视频、音频、页面截图、商品信息、评论回复、搜索结果、推荐流、直播上下文、结构化字段等任意电商相关对象。
   - 明确输入字段：原始内容、截图、视频帧、OCR 文本、ASR 文本、query、用户画像、商品信息、模型标签、系统识别结果、人工标注等。

3. **提取或归纳规则**
   - 有规则文档时，读取 `references/rule-extraction-guide.md`，提取规则体系、判定条件、白名单、豁免、优先级、输出要求。
   - 无规则文档时，读取 `references/example-rule-induction-guide.md`，从正反例中归纳定义、命中条件、排除条件、边界说明和不确定情况处理。
   - 始终区分「最终输出标签」与「内部判定依据」。

4. **套用统一 prompt 框架**
   - 不按场景选择独立模板，不维护“图片模板”“搜索模板”“直播模板”等场景模板。
   - 使用 `references/unified-prompt-template.md` 中的唯一统一模板，通过变量槽位适配具体任务。
   - 明确规则来源：`规则文档` 或 `正反例归纳`。

5. **生成 prompt**
   - 默认章节顺序：`# 角色` → `# 背景说明` → `# 巡检任务` → `# 输入信息` → `# 规则定义` → `# 判定原则` → `# 质检执行步骤` → `# 输出要求`。
   - 规则描述优先使用「定义 / 判定条件 / 关键识别点 / 示例 / 白名单豁免 / 优先级 / 边界说明」。
   - 输出要求必须明确字段名、类型、枚举值、布尔值、null 规则。

6. **交付前自检**
   - 检查是否正确走了规则文档链路或正反例归纳链路。
   - 检查是否把示例场景误当作能力边界。
   - 检查无规则文档时是否从正反例抽象出了可迁移规则，而不是复述样本。
   - 检查是否适配当前任务的实际输入模态。
   - 检查是否遗漏白名单、豁免、优先级、冲突处理和不确定情况处理。

## 需要读取的参考资料

- 需要模仿历史 prompt 写法时，读取 `references/prompt-style-guide.md`。
- 有规则文档时，读取 `references/rule-extraction-guide.md`。
- 无规则文档、只有正反例时，读取 `references/example-rule-induction-guide.md`。
- 需要生成 prompt 时，读取 `references/unified-prompt-template.md`。
- 交付前必须参考 `references/quality-checklist.md` 做自检。

## 交付原则

- 优先直接交付完整 prompt，不只给思路。
- 不为每个电商场景单独套用或维护模板；始终使用统一模板适配。
- 有规则文档时，不遗漏规则中的白名单、豁免、反例、优先级和冲突处理。
- 无规则文档时，不只复述正反例，要抽象出可迁移的规则定义、命中条件、排除条件和边界说明。
- 不把内部判定依据误写成最终输出字段。
- 输出字段保持稳定，适合妙鉴平台解析。
- 对不确定输入字段使用占位符，例如 `{content}`、`{image}`、`{video}`、`{audio}`、`{ocr_text}`、`{asr_text}`、`{structured_fields}`、`{positive_examples}`、`{negative_examples}`、`{model_result}`。
