## 1. 写在前面

随着电商内容、搜索推荐、直播与短视频等业务规模持续增长，传统依赖人工抽检的质量巡检方式已经难以满足业务快速迭代、问题快速发现和策略持续优化的诉求。人工巡检通常存在以下痛点：

- **覆盖不足**：人工只能抽样检查，难以覆盖海量内容、商品、推荐结果和直播场景。
- **效率有限**：问题发现依赖人工排查，周期长、反馈慢。
- **标准不稳定**：不同人员对规则理解可能存在差异，导致判定标准不一致。
- **能力难沉淀**：巡检经验、规则、Prompt、模型优化记录分散，难以复用和迭代。
- **闭环不完整**：发现问题后，人工纠错、样本沉淀、策略优化、模型学习之间缺少平台化链路。

**妙鉴平台**定位为一套 AI Native 的智能巡检平台，通过「数据圈选 → 巡检能力建设 → 巡检任务配置 → AI 自动执行 → 结果分析 → 人工纠错 → AI 自学习」的完整链路，实现巡检工作的自动化、智能化和平台化。

## 2. 平台定位与核心价值

### 2.1 平台定位

妙鉴平台是面向多业务场景的智能巡检基础设施，主要用于承载搜索推荐、商品推荐、直播回评、视频流推荐等场景下的 AI 巡检工作。平台既支持业务人员快速发起巡检任务，也支持研发和算法同学建设、管理和迭代巡检能力。

平台核心目标是：

> 将业务质量巡检从「人工抽检」升级为「AI 自动巡检 + 人工反馈 + 能力持续学习」的闭环体系。

### 2.2 核心价值

| 价值方向 | 说明 |
|---|---|
| 自动化 | 支持 Excel 上传、SQL 取数、定时巡检、批量执行，减少人工重复操作。 |
| 智能化 | 支持 Prompt 驱动等 AI 能力接入，自动识别异常 case。 |
| 白盒化 | 巡检能力可查看输入输出字段、Prompt 配置、使用示例和运行结果，方便理解和评估。 |
| 平台化 | 统一管理巡检任务、巡检结果、巡检能力和自学习样本，降低跨团队协作成本。 |
| 可迭代 | 支持人工纠错进入样本池，后续用于评审、训练、Prompt 优化和能力版本迭代。 |

## 3. 平台整体架构

妙鉴平台整体可抽象为七层链路：

```text
数据接入层 → 数据预处理层 → 巡检能力层 → 任务调度层 → 结果分析层 → 人工反馈层 → AI 自学习闭环
```

### 3.1 业务流程总览

1. **圈选巡检对象**：从已有数据集选择，或通过 Excel / SQL 创建新的巡检对象。
2. **数据预处理**：按业务需要进行视频抽帧、ASR 语音识别、自定义字段处理等。
3. **选择巡检能力**：选择已建设且公开的 AI 巡检能力，并查看能力详情。
4. **参数与字段对齐**：根据巡检能力的 Input / Output 字段，确保数据集字段与能力入参语义一致。
5. **配置巡检任务**：填写任务名称、数据集名称、结果分类字段、执行策略、消息推送等。
6. **创建并执行**：平台创建巡检任务并自动执行。
7. **查看巡检结果**：从综合看板、批次列表、批次详情、分组巡检记录等层级分析结果。
8. **人工判定与纠错**：对 AI 判定结果进行人工确认或纠错。
9. **进入自学习样本池**：纠错样本沉淀到样本池，等待评审和后续模型/策略迭代。

## 5. 核心模块介绍

## 5.1 巡检任务

「巡检任务」是平台的任务发起入口，用于管理已有巡检任务，并创建一次性或周期性自动巡检。

线上任务页面的标题为 **巡检任务**，副标题为「管理巡检任务，支持多周期自动巡检」。页面右上角提供：

- 搜索框：搜索任务名称。
- 筛选按钮：按条件过滤任务。
- 新建巡检任务按钮：进入新建任务页。

### 5.1.1 任务列表

任务列表用于管理所有已创建的巡检任务，线上表格字段包括：

- 任务名称。
- 巡检对象。
- 状态。
- 最近执行。
- 操作。

任务名称可点击，操作区提供：

- **查看详情**：查看任务配置与执行信息。
- **查看结果**：跳转至该任务关联的巡检结果。

线上示例任务包括：

- 综搜出卡巡检_商品词2.0。
- 综搜出卡巡检_寻址词2.0。
- 逛览总结巡检。
- 逛览发现性巡检。
- 逛览匹配性巡检。
- 货架高爆图片质量巡检。
- 综搜推荐巡检_商品词。
- 综搜推荐巡检_寻址词。
- 泛货架推荐商品巡检。

任务列表底部支持分页，并显示总条数。

任务状态说明：

| 状态 | 含义 |
|---|---|
| 待执行 | 任务已创建，但尚未开始执行。 |
| 执行中 | 任务正在进行数据处理或 AI 巡检。 |
| 已完成 | 当前批次巡检已执行完成。 |
| 失败 | 任务执行异常，需要排查数据、能力或系统原因。 |

### 5.1.2 新建巡检任务

新建巡检任务线上采用纵向表单布局，主要包含 5 个板块：

#### 01 基本信息

用于填写任务名称。

| 字段 | 说明 |
|---|---|
| 任务名称 | 必填，填写本次巡检任务名称。 |

#### 02 圈选巡检对象

平台支持两种方式圈选巡检对象：

| 数据来源 | 说明 | 适用场景 |
|---|---|---|
| Excel 上传 | 上传本地 `.xlsx`、`.xls`、`.csv` 文件。 | 临时巡检、业务人工整理数据。 |
| SQL 取数 | 编写 SQL 从数据仓库或业务库取数。 | 需要自动化、周期化、批量化获取数据。 |

线上 Excel 上传区包含：

| 字段 / 区域 | 说明 |
|---|---|
| 数据集名称 | 必填，可由上传文件自动读取，也可手动填写。 |
| 结果分类字段 | 必填，用于结果分组，可手动填写分类列名。 |
| 文件上传区 | 支持点击或拖拽文件上传。 |

上传说明：支持 `.xlsx`、`.xls`、`.csv` 文件，单个文件不超过 100MB。

Excel 上传支持：

- 数据集名称配置。
- 拖拽上传或点击选择文件。
- 支持 `.xlsx`、`.xls`、`.csv`。
- 文件大小建议不超过 100MB。
- 上传后可进行数据预览。

SQL 取数支持：

- SQL 语句编辑。
- SQL 语法验证。
- 数据量预估。
- 数据预览。
- 可使用日期等参数化变量。

#### 数据预处理

在圈选数据后，可以按需启用数据预处理。线上开关文案为 **启用数据预处理**，说明为「勾选后对上传数据进行视频抽帧、ASR 处理等操作」。

| 预处理类型 | 配置项 | 用途 |
|---|---|---|
| 视频抽帧 | 直播/短视频场景、ID 列、时间列、最大抽帧数 | 从视频或直播内容中抽取关键画面，供多模态模型分析。 |
| ASR 语音识别 | 语言、采样率、输出字段名 | 将直播或视频语音转成文本，辅助内容理解。 |
| 其他处理 | 自定义标签与字段映射 | 支持业务自定义预处理字段，增强泛化能力。 |

预处理完成后，平台会展示数据预览，默认最多展示前 10 条记录，帮助用户确认数据是否符合预期。

#### 03 选择巡检能力

线上通过下拉选择框选择本次任务要使用的巡检能力。选择前需要确保能力已在「巡检能力」模块中创建并公开。

#### 04 执行策略

平台支持两种执行策略：

| 执行策略 | 说明 |
|---|---|
| 单次执行 | 立即跑一次，适用于临时巡检、试跑和一次性专项排查。 |
| 定时自动巡检 | 按周期自动触发，适用于日常质量监控和周期性巡检。 |

#### 05 任务完成消息推送

线上新建任务页底部支持配置任务完成后的消息推送对象：

| 字段 | 说明 |
|---|---|
| 接收人工号 | 可选，填写接收通知的用户工号。 |
| 群聊 ID | 可选，填写 KIM 群聊 ID。 |
| 业务负责人 | 可选，填写业务负责人相关信息。 |
| 任务负责人 | 可选，填写任务负责人相关信息。 |

配置完成后，点击 **创建任务** 提交；也可以点击 **取消** 返回。

## 5.2 巡检结果

「巡检结果」用于查看平台所有巡检批次、分组记录和单个 Case 的详细分析结果。

### 5.2.1 首页指标

结果首页顶部展示 3 个核心指标卡：

| 指标 | 说明 |
|---|---|
| 巡检业务类型 | 已接入的业务类型数量。 |
| 进行中 | 当前预处理或巡检中的任务数量。 |
| 周巡检总量 | 本周巡检任务数。 |

线上示例中，巡检业务类型为 1，进行中为 0，周巡检总量为 1。

### 5.2.2 综合看板

巡检结果页提供 **综合看板**，用于从多级统计维度查看巡检结果分布。

线上看板支持：

- 按批次 ID 搜索，支持逗号分隔多个批次。
- 按日期范围筛选。
- 按任务 / 巡检类型下拉筛选。
- 展开或收起一级、二级、三级统计维度。
- 查看 UID 数量、UID 占比、ItemId 数量、ItemId 占比。
- 使用进度条直观展示占比。

综合看板示例维度包括：

| 一级统计维度 | 二级统计维度 | 三级统计维度 / 错误类型 |
|---|---|---|
| 未发现问题 | U0-滞客、U0-流失、U1、U2、U3、U4、U4+、未知 | — |
| 发现问题 | U3、U4、U4+、U2、U1、U0-流失等 | 连坑重复、相邻重复 |

该看板适合管理者和业务方快速查看问题分布、影响 UID / ItemId 规模，并定位异常集中维度。

### 5.2.3 批次列表

批次列表以任务执行批次为维度展示巡检结果，线上字段包括：

- 批次 ID。
- 任务名称。
- 状态。
- 巡检能力。
- 检出 / 总条数。
- 触发方式。
- 创建时间。
- 耗时。
- 操作。

批次列表支持：

- 按任务下拉筛选。
- 按状态筛选。
- 重置筛选。
- 查看结果。

线上示例批次信息包括：

- 任务名称：泛货架推荐商品巡检。
- 巡检能力：商品相似度匹配。
- 触发方式：单次 / 定时。
- 检出 / 总条数：如 482 / 3332、754 / 3332。
- 耗时：如 607.5min、485.7min。

常用操作：

- **查看结果**：进入该批次的统计详情和分组结果。
- **重置筛选**：清空当前筛选条件。
- **刷新**：重新加载批次列表。

### 5.2.3 分组记录

分组记录以单个分组 / Case 为维度展示结果，字段包括：

- 记录 ID。
- 所属批次。
- 时间。
- Case。
- 策略。
- 状态。
- 判定结果。
- 耗时 / 置信度。
- 发起人。
- 操作：详情。

支持按判定筛选：

- 存在问题。
- 未发现问题。

### 5.2.4 批次详情

批次详情用于查看某一次巡检执行的完整概况。线上批次详情包含：

#### 基础信息

| 字段 | 说明 |
|---|---|
| 发起人 | 本次批次发起人，例如 `wujunjie03`。 |
| 批次 | 批次 ID，例如 `235`。 |
| 创建时间 | 批次创建时间，例如 `2026-07-14 10:15`。 |
| 数据版本 | 本次巡检使用的数据版本，例如 `880`。 |
| 分组字段 | 本次结果聚合字段，例如 `t1.user_id`。 |
| 策略配置快照 | 本次批次使用的巡检能力，例如「商品相似度匹配」。 |

#### 汇总指标

线上批次详情展示 4 个汇总卡片：

| 指标 | 说明 | 示例 |
|---|---|---|
| 总分组 | 本批次分组总数。 | 3332 |
| 检出问题 | 被识别为存在问题的分组数。 | 482 |
| 执行失败 | 执行失败的分组数。 | 0 |
| 未发现问题 | 判定为正常的分组数。 | 2849 |

#### 综合看板

批次详情页同样提供综合看板，可在单批次范围内查看问题 / 非问题分布，并支持导出 CSV。

#### 分组结果

分组结果区展示该批次下所有分组记录，支持：

- 全部。
- 存在问题。
- 未发现问题。
- 待人工审查。
- 执行失败。

线上字段包括：条目、巡检记录 ID、分组字段、巡检能力、状态、判定、错误类型、耗时、操作。支持导出巡检结果、刷新以及进入单条详情。

### 5.2.5 分组巡检记录详情

分组巡检记录详情是巡检结果的最小分析单元，线上标题为 **分组巡检记录**，主要展示单个分组的判定结果、人工判定和策略明细。

页面顶部展示：

- 分组字段，例如 `1003884877`。
- 状态，例如已完成。
- 发起人。
- 耗时，例如 `6.7s`。
- 创建时间。
- 完成时间。

判定区域展示：

- 综合判定，例如「不存在问题」。
- 策略检出数量，例如 `0/1 策略检出`。

人工判定支持：

| 操作 | 说明 |
|---|---|
| 存在问题 | 人工确认当前分组存在问题。 |
| 不存在问题 | 人工确认当前分组不存在问题。 |
| 纠错 | 提交纠错反馈。 |
| 判定原因 | 可选填写，用于说明人工判定依据。 |

内容分析区包括：

- Feed 模拟器：展示当前分组下的商品 / 内容列表，可在「仅问题 / 全部」之间切换。
- 策略详情：支持切换不同策略查看详情，例如 `feed相邻重复`、`feed连坑重复`。
- 策略命中结果：展示是否命中相似或重复问题，例如「未命中任何相邻相似」。

## 5.3 巡检能力

「巡检能力」是妙鉴平台的 AI 能力资产管理中心，主要服务于研发、算法和平台管理员。

### 5.3.1 页面布局

线上能力页采用左右分栏布局：

- **左侧能力列表**：展示能力卡片，支持搜索能力名称、筛选、新建能力、公开开关、编辑入口。
- **右侧能力详情**：以编号区块展示能力基础信息、Input / Output 字段、使用示例、运行结果等。

能力列表卡片展示：

- 能力名称。
- 公开状态开关。
- 编辑入口。
- 能力类型标签，例如 `Prompt`。
- 业务类型标签，例如 `搜索推荐`、`商品推荐`。
- 版本号，例如 `v7`、`v6`。
- 更新时间，例如 `2026-07-14`。

线上示例能力包括：

- 综搜出卡巡检寻址词2.0。
- 逛览发现性巡检。
- 逛览匹配性巡检。
- 综搜出卡巡检商品词2.0。
- 商品相似度匹配。

### 5.3.2 能力类型

当前线上平台重点支持 **Prompt 驱动** 能力建设。Prompt 驱动能力用于通过大模型、Prompt 模板和结构化输入输出字段完成业务巡检判断。

平台能力列表中，能力类型以标签展示，例如 `Prompt`；业务类型以标签展示，例如 `搜索推荐`、`商品推荐`。

### 5.3.3 新建巡检能力

线上点击 **新建能力** 后，会打开「新建巡检能力」弹窗。弹窗主要分为 3 个区域：

#### 基础信息

| 字段 | 说明 |
|---|---|
| 巡检名称 | 必填，填写能力展示名称。 |
| 能力标识 | 必填，唯一标识，创建后不可改，例如 `image_text_consistency`。 |
| 业务类型 | 必填，可输入或选择业务类型。 |
| 能力类型 | 必填，当前可选择 Prompt 驱动。 |
| 描述 | 选填，简要描述该能力用途。 |

#### Prompt 驱动配置

| 字段 | 说明 |
|---|---|
| 模型 | 选择模型，例如 `deploy/QWEN3_5`。 |
| Prompt 模板 | 必填，用于定义模型角色、输入字段、场景和判断标准。 |
| 一键提取参数 | 根据 Prompt 模板自动抽取输入 / 输出参数，降低配置成本。 |

Prompt 模板示例结构：

```text
你是一名商品巡检专家，给定商品信息如下：
商品ID：{{item_id}}
场景：{{scene}}
请判断该商品是否符合平台规范……
```

#### 输入 / 输出参数

| 字段 | 说明 |

| 示例值 | 示例输入值。 |

| 字段 | 说明 |
|---|---|

弹窗支持继续添加输入参数和输出参数。

## 5.4 AI 自学习

「AI自学习」用于承接人工纠错样本，形成模型和策略持续优化闭环。

### 5.4.1 自学习流程

平台中的自学习流程为：

```text
发现误判 → 提交纠错 → 进入样本池 → 等待评审 → 模型迭代更新
```

### 5.4.2 样本来源

样本主要来自「巡检结果」中的分组巡检记录详情页。当用户发现 AI 判定错误时，可以点击 **纠错**，提交正确标注和原因。提交后，该样本会进入自学习样本池。

| 说明原因 | 人工填写的纠错原因。 |

### 5.4.4 样本状态

| 状态 | 含义 |
|---|---|

## 6. 典型使用流程

## 6.1 业务方发起一次巡检

适用角色：业务、运营、产品、测试。

1. 进入 **巡检任务**。
2. 点击 **新建巡检任务**。
3. 在 **基本信息** 中填写任务名称。
4. 在 **圈选巡检对象** 中选择数据来源：
   - Excel 上传；或
   - SQL 取数。
5. 填写数据集名称和结果分类字段。
6. 上传文件或填写 SQL，并按需启用数据预处理。
7. 在 **选择巡检能力** 中选择本次任务要使用的能力。
8. 在 **执行策略** 中选择：
   - 单次执行：立即跑一次；或
   - 定时自动巡检：按周期自动触发。
9. 如需通知相关人员，在 **任务完成消息推送** 中填写接收人工号、群聊 ID、业务负责人、任务负责人。
10. 点击 **创建任务**。
11. 进入 **巡检结果** 查看批次执行结果。

## 6.2 查看并分析巡检结果

1. 进入 **巡检结果**。
2. 查看顶部指标卡，包括巡检业务类型、进行中任务数、周巡检总量。
3. 在 **综合看板** 中按批次 ID、日期范围、任务类型筛选，查看 UID / ItemId 数量与占比。
4. 在 **批次列表** 中找到目标批次。
5. 点击 **查看结果**，进入批次详情页。
6. 查看批次基础信息、策略配置快照、总分组、检出问题、执行失败、未发现问题等指标。
7. 在分组结果中按「全部 / 存在问题 / 未发现问题 / 待人工审查 / 执行失败」筛选记录。
8. 点击单条分组记录的 **详情**。
9. 查看分组字段、综合判定、人工判定区、Feed 模拟器和策略命中详情。
10. 如 AI 判定错误，可进行人工判定或纠错。

## 6.3 提交纠错并进入自学习

1. 在 **巡检结果 → 分组巡检记录详情** 中发现 AI 误判。
2. 在人工判定区域选择 **存在问题**、**不存在问题** 或 **纠错**。
3. 在判定原因中填写可选说明。
4. 提交后，纠错信息可进入后续样本评审与自学习流程。
5. 进入 **AI自学习** 查看样本池和评审状态。

## 6.4 建设一个新的巡检能力

适用角色：研发、算法、平台管理员。

1. 进入 **巡检能力**。
2. 点击左侧能力列表顶部的 **新建能力**。
3. 在「基础信息」中填写巡检名称、能力标识、业务类型、能力类型和描述。
4. 在「Prompt 驱动配置」中选择模型，例如 `deploy/QWEN3_5`。
5. 编写 Prompt 模板，定义模型角色、输入字段、场景和判断标准。
6. 如 Prompt 已包含结构化变量，可点击 **一键提取参数**。
7. 配置输入参数和输出参数，包括参数名、类型 / 字段角色、名称和示例值。
8. 保存后，在能力列表中确认能力版本、公开状态和业务类型标签。
9. 如需调整，可点击能力卡片上的 **编辑**。
10. 能力公开后，业务方即可在「新建巡检任务」中选择该能力。

## 7. 典型业务场景

### 7.1 搜索推荐巡检

用于检查搜索结果是否满足业务承接标准。例如用户搜索强电商意图词时，首屏是否出现正确的电商承接卡片。

可能使用能力：

- Prompt 驱动多模态分析。
- 结构化 Input / Output 字段归因。
- 多级统计维度输出。

常见输入：

- 搜索关键词。
- 截图 URL。
- 搜索结果列表。
- 曝光位置。

常见输出：

- 是否异常。
- 异常类型。
- 判定原因。
- 置信度。

### 7.2 商品相似度巡检

用于识别商品推荐流中相似商品是否过度集中，避免用户体验下降。

可能使用能力：

- 商品相似度匹配。
- Prompt 驱动规则判断。
- 多级错误类型归因。

常见配置：

- 相似度阈值。
- 最大坑位数。
- 步长。
- 最小连续对数。
- RPC 超时时间。

### 7.3 直播回评巡检

用于对直播内容进行回放、抽帧、ASR 和模型分析，识别内容质量问题。

可能使用能力：

- Prompt 多模态判断。
- 视频抽帧 / ASR 预处理。
- 结构化异常类型输出。

常见输入：

- 直播间 ID。
- 曝光时间。
- 抽帧结果。
- ASR 文本。

### 7.4 视频流推荐巡检

用于检查推荐流内容是否满足质量、合规、重复度或业务策略要求。

可能使用能力：

- Prompt 驱动。
- 结构化字段输入输出。
- 相似度 / 重复度策略判断。

## 10. 最佳实践

### 10.1 任务命名

建议任务名称包含：业务场景 + 巡检目标 + 时间周期。

示例：

- 搜索推荐承接巡检-2026W21。
- 商品相似度巡检-女装类目-每日。
- 直播回评质量巡检-大促专项。

### 10.2 结果分类字段选择

结果分类字段决定批次详情中的分组粒度，应选择业务上可解释、可聚合的字段，例如：

- 搜索场景：`keyword`。
- 商品场景：`category`、`spu_id`、`seller_id`。
- 直播场景：`live_id`、`anchor_id`。
- 视频场景：`item_id`、`author_id`。

### 10.3 输入输出字段配置

输入输出字段建议遵循：

- Input 字段应与任务数据中的字段语义一致。
- Output 字段应覆盖最终判定、错误类型、统计维度和模型分析。
- 关键判定字段建议固定命名，例如 `is_anomaly`。
- 统计维度字段建议命名清晰，便于综合看板归因。

### 10.4 巡检能力建设

建设能力时建议同时维护：

- 能力定义。
- Prompt 模板。
- Input / Output 参数说明。
- 使用示例。
- 输出统计维度。
- 版本记录和变更说明。

### 10.5 人工纠错

提交纠错时建议填写清晰原因，例如：

- 「AI 未识别图片中的违禁 logo」。
- 「该商品展示正常，AI 将普通促销误判为夸大宣传」。
- 「截图中实际没有电商承接卡片，属于漏检」。

高质量纠错原因可以帮助后续评审、训练和 Prompt 优化。

## 11. 常见问题

### Q1：没有合适的巡检能力怎么办？

可以先联系平台研发或算法同学，在「巡检能力」中建设新能力。若已有类似能力，也可以复用后修改场景、输入输出和业务规则。

### Q2：为什么要配置 Input / Output 字段？

Input 字段决定能力运行时接收哪些数据；Output 字段决定模型输出如何被平台解析、展示和统计。字段配置越清晰，后续任务创建、结果归因和问题分析越稳定。

### Q3：为什么要配置结果分类字段？

结果分类字段决定巡检结果如何分组展示。例如搜索场景通常按 `keyword` 分组，商品场景可按 `category` 或 `spu_id` 分组。选择合理的分类字段有助于快速定位问题范围。

### Q4：定时巡检适合什么场景？

适合周期性质量监控，例如每日搜索推荐巡检、每周商品相似度巡检、固定大促期间直播回评巡检等。

### Q5：人工纠错后会立即影响模型吗？

不会立即影响。纠错样本会先进入 AI 自学习样本池，经过评审后才会进入训练集、评测集、Prompt 优化或能力版本迭代流程。

### Q6：批次详情和分组记录有什么区别？

批次详情是一次任务执行的整体视角，适合看汇总和分组列表；分组巡检记录是单个分组视角，适合看综合判定、人工判定、Feed 模拟器和策略命中详情。

### Q7：巡检能力中的 Input / Output 如何理解？

- Input 是能力运行所需的输入字段，例如 `url`、`keyword`、`category`。
- Output 是能力运行后的结构化结果，例如 `is_anomaly`、`error_type`、多级统计维度和 `visual_evidence`。
- Output 字段会进一步支撑巡检结果页中的综合看板、问题归因和明细展示。
