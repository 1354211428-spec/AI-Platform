# AI巡检平台搭建PRD

## 背景

电商内容平台的规模化增长，传统依赖人工抽检的巡检模式已经无法满足业务快速迭代需求。因此，需要建设一套 AI Native 的智能巡检平台，通过「自动取数 + 创建巡检 + 策略管理 + 结果闭环 + AI自主学习」实现巡检自动化、智能化、平台化。

## 核心模块详细说明

### 1. 圈选巡检对象（取数）——该板块命名为数据库

- **支持 excel 直接上传离线数据**
  - 上传方式支持：拖拽上传、本地选择文件上传、模版下载
  - 上传校验：文件格式、文件大小、字段完整性、数据类型（自动校验）
  - 字段标准化配置：上传后平台自动识别字段，定义字段类型-文本/数字/日期/标签/URL等

- **支持上传 SQL 自动 hive 表取数：**
  - 上传SQL → SQL验证（是否有误）→ 资源预估（预估数据量与时间）→ 数据预览 → 选择抽样方式（全量/抽样）→ 执行取数 → 数据入库
  - **SQL编辑能力**：支持SQL编辑器、SQL高亮、自动补全、简单修改、参数化变量
  - **数据预览**：执行前自动展示前10条数据、字段类型，用于确定取数是否符合预期
  - **资源预估**：数据规模预估、执行耗时预估、超大数据量告警等
  - **数据抽样**：可选择全量输出/固定条数/分层抽样等
  - **自动化调度能力**：每日执行/每周执行/自定义周期。调度监控：调度成功率、最近执行时间、异常报警、执行日记等
  - **AI SQL优化**：SQL语法纠错、自动补全where条件、自动增加limit、SQL性能优化建议（检测到当前SQL存在全表扫描风险，建议增加日期过滤条件等）、自动生成取数模版等

### 2. 巡检平台（发起巡检）

- **可支持场景下拉选择**：视频流推荐、B端直播间、搜索推荐、商品推荐等等
- **选择数据**：先选择数据库里的数据、选完数据后对数据预处理，比如直播间和视频业务需要根据视频url对视频截图来获取拖画面，因此需要输入直播间ID、时间戳、抽帧数等等；搜推业务，要根据业务方所给的关键词，在快手搜索栏搜索后截图，获取截图的url等等。做完以上预处理后，需要有个自动的数据预览，显示数据的前10行
- **配置巡检**：支持自定义巡检名、可从数据集中选择分组字段、可以给case加前缀、模型选择等
- **创建巡检规则**：左侧可展示规则名，右侧为规则的详情展示，规则的详情页面可以对里面的内容进行编辑修改

#### 2.1 定时自动巡检——周期配置

勾选"启用定时自动巡检"后，点击"设置周期"按钮，弹出**自定义周期**配置弹窗：

**重复频率区域**

| 频率单位 | 交互控件 | 说明 |
|----------|----------|------|
| 每 N 日 | 数字步进 + "日"下拉 | 如"每 1 日"即每天执行 |
| 每 N 周 | 数字步进 + "周"下拉 + 星期多选按钮组（一二三四五六日） | 如"每 1 周，选周三、周五" |
| 每 N 月 | 数字步进 + "月"下拉 + "每个月的第 X 天"下拉（1-28） | 如"每 1 月，第 20 天" |

**结束时间区域**（二选一）

- ⚪ **重复 N 次后**：下拉选择执行次数（10/30/60/90/180/不限），默认 30 次
- ⚪ **到指定日期**：日期输入框（格式 YYYY-MM-DD）

**执行时间**：HH:MM 时间选择，默认 02:00

**弹窗底部**：取消 / 确定

**周期摘要**（弹窗确认后，在创建任务页内联展示）：

> 📅 每 1 周，周三、周五 02:00 执行，重复 30 次

**数据结构（`schedule_config` JSON 字段）**

```json
{
  "freq_unit":     "day" | "week" | "month",
  "freq_interval": 1,
  "week_days":     [2, 4],      // 0=周一…6=周日，仅 freq_unit=week 时有效
  "month_day":     20,          // 仅 freq_unit=month 时有效
  "exec_time":     "02:00",
  "end_mode":      "count" | "date",
  "end_count":     30,
  "end_date":      "2026-05-30"
}
```

**`schedule_period` 枚举映射**

| freq_unit | schedule_period |
|-----------|----------------|
| day | daily |
| week | weekly |
| month | monthly |

### 3. 巡检结果概览（含巡检概览看板）

#### 3.1 巡检概览（管理层看板）

与"巡检结果"并列为横向一级 Tab，供管理层快速了解平台整体巡检健康状态。

**核心指标卡（6个）**

| 指标 | 计算逻辑 |
|------|---------|
| 累计巡检 Case 数 | 所有版本 total_count 汇总 |
| 巡检覆盖率 | 已巡检任务 / 总任务数（%） |
| 异常 Case 占比 | 全局 badcase_count / total_count |
| 平均巡检耗时 | 版本执行时长均值（秒） |
| 进行中任务数 | status=running 的任务数 |
| 本周新增任务 | 7天内创建的任务数 |

**同环比趋势**

- 周环比：本周 vs 上周 badcase 率变化（%）
- 月同比：本月 vs 上月巡检总量变化

**近7日每日巡检趋势**（简单条形图，按日展示 total / badcase 数量）

**任务健康状态表**（列：任务名、最新版本、badcase率、执行时间、状态）

**后端接口**：`GET /api/v1/result/overview-stats`

#### 3.2 巡检结果（原有内容）

- **巡检结果概览**：可用看板的格式，顶部为指标大卡，用来展示核心数据，如巡检case数、badcase数、badcase率、误判率（人工纠错比例）、巡检评分等等核心指标
- **巡检详情**：点击详情后可查看每个case的详细情况，以及判定原因、选择的策略、命中的规则、截图预览（直播/视频）、操作日志等等
- **人工反馈**：人工可驳回badcase并给明原因、平台自学习抽象规则，自动避免下次同情况发生
- **AI自学习**：标准集进入训练集 → Prompt优化 → 模型微调 → 策略自动优化
- **日记系统**：记录SQL执行日记｜AI分析日记｜Prompt修改日志｜用户操作日志

### 4. AI巡检能力

#### 4.1 整体结构

两个横向 Tab，无场景分类子 Tab：

- **创建巡检能力（研发视角）**：可新建、编辑、发布巡检能力
- **巡检能力预览（业务视角）**：只读，通过场景下拉筛选

#### 4.2 创建巡检能力（研发视角）

新建巡检能力 Modal 字段：
- 巡检名称（必填，唯一标识）
- 场景（必填，文本自由输入）
- 模板类型（必填，下拉）
- 描述（选填）

**5种模板类型及其模板配置：**

| 模板类型 | 配置字段 |
|---------|---------|
| AI智能分析 | 模型（下拉：Gemini 3.1 Pro/GPT-4o等）、分析指令（大文本） |
| 内容频控 | 跨刷同item最小间隔、跨刷同spu最小间隔、跨刷同类目最小间隔、同作者冷却时间(秒) |
| 商品相似度 | 子规则（ADJACENT/STEPPED 二选一）、相似度阈值、最大坑位数、步长、最小连续对数、单次RPC超时(ms) |
| 规则阈值 | 目标字段、备选字段、匹配值、下限阈值、上限阈值、低/高于阈值标签、正常标签 |
| 工作流 | 简要说明 + 工作流节点接入区 |

#### 4.3 巡检能力预览（业务视角）

- 场景下拉：视频流推荐 / B端直播间 / 搜索推荐 / 商品推荐
- 按场景过滤能力列表，只读展示能力详情
- 无新建/编辑/发布操作
- **v3.1 新增**：点击能力卡片可弹出白盒详情弹窗，展示格式与"新建巡检能力"弹窗一致（按模板类型渲染对应配置字段，只读）

## v11.0 改动说明

### 改动：巡检任务新建流程新增「字段映射」板块

**背景**：巡检对象（数据集）上传后的字段名与巡检能力要求的 Input 字段名可能不一致，需要提供字段映射配置，让用户将数据集字段一一映射到巡检能力的输入字段上，实现统一。

**涉及文件**：`backend/src/database.py`、`backend/src/routes_task.py`、`frontend/src/task.js`、`frontend/src/style.css`

#### 数据模型

`inspection_task` 表新增字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| field_mapping | TEXT | JSON 格式，记录 能力Input字段→数据集字段 的映射关系 |

示例：`{request_id: req_id, timestamp: ts, log_content: content}`

#### 接口变化

- `POST /api/v1/tasks`：新增 `field_mapping` 字段（dict）
- `GET /api/v1/tasks/<id>`：响应新增 `field_mapping`（解析为 dict）

#### 前端变化

新建巡检任务页面：在「02 选择巡检能力」之后插入「03 字段映射」板块，原「03 配置巡检」变为「04 配置巡检」。

字段映射板块设计：
- 标题行：左列「巡检能力 Input 字段」，右列「数据集字段映射」
- 每行展示一个能力 Input 字段，右侧提供下拉+输入框：
  - 若已选择数据集且数据集有 field_schema，则下拉列出所有可用字段供快速选择
  - 支持手动输入字段名
- 已选择能力后才展示映射表；未选择能力时显示提示
- `form.fieldMapping` 存储映射结果

- 版本号 v12 → v13
