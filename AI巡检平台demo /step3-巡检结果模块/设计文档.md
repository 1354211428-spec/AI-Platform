# 巡检结果模块 设计文档 v3.0

> v3.0 重构：首页改为指标大卡 + 双Tab（批次列表/分组记录）；新增批次详情页（图3）、分组巡检详情页（图4）。

---

## 1. 功能定位

- 汇总展示全平台所有巡检批次和分组记录
- 支持按批次维度和分组记录维度分别查看
- 下钻查看单个批次的详细统计和分组结果
- 再次下钻查看单条分组的模型分析、判定原因、命中能力

---

## 2. 页面结构 & 路由

```
view: 'dashboard'    → 首页（指标大卡 + 双Tab）
view: 'batch-detail' → 批次详情页（图3）
view: 'case-detail'  → 分组巡检记录详情页（图4）
```

## 3. 首页（dashboard）

### 3.1 指标大卡区（顶部）

| 卡片 | 数据来源 |
|------|---------|
| 当日巡检总数 | `daily-stats.total_today` |
| 正在进行巡检数 | `daily-stats.running_count` |
| 今日已完成 | `daily-stats.done_today` |
| 今日问题数 | `daily-stats.badcase_today` |

**API：** `GET /api/v1/result/daily-stats`

### 3.2 双Tab区

**Tab1 — 批次列表（对应图1）：**

```
批次列表  N 条                                    [🔄 刷新]
─────────────────────────────────────────────────────────
批次名称 | 状态  | 分组字段 | 巡检场景 | 策略
        | 分组执行结果 | 发起人 | 创建时间 | 耗时 | 操作
─────────────────────────────────────────────────────────
综搜推荐 | 已完成 | keyword | 搜索推荐 | 综搜商品检测
        | 49组·9检出 | wujunjie03 | 2026/5/19 | 18.9min
        |                                    | [详情] [复用配置]
```

字段映射：
- 批次名称 = `inspection_task.name`
- 状态 = `inspection_version.status`（最新版本）
- 分组字段 = `inspection_task.group_field`
- 策略 = 使用的巡检能力名称（chips）
- 分组执行结果 = `N组·M检出`（badcase）
- 耗时 = 版本执行耗时（min/s）

**Tab2 — 分组记录（对应图2）：**

```
分组记录  N 条      [判定筛选▼]  [策略筛选▼]   [🔄 刷新]
─────────────────────────────────────────────────────────
记录ID | 所属批次 | 时间 | Case | 策略 | 状态 | 判定 | 耗时 | 发起人 | 操作
─────────────────────────────────────────────────────────
r_xxx  | b_xxx   | ...  | 202605… | 1586… | 已完成 | 存在问题 | 124.4s |   | [详情]
```

字段映射：
- 记录ID = `inspection_case.case_id`（短格式）
- 所属批次 = 关联 `inspection_version` 的 batch ID
- 判定 = `ai_result`（存在问题/未发现问题）

## 6. 数据接口设计

### 新增接口

```
GET /api/v1/result/daily-stats
Response: {
  total_today: int,        # 当日巡检总数（版本）
  running_count: int,      # 正在执行的任务数
  done_today: int,         # 今日完成
  badcase_today: int,      # 今日发现问题数
}
```

### 复用接口

| 接口 | 用途 |
|------|------|
| `GET /api/v1/result/task-summary` | 批次列表Tab |
| `GET /api/v1/result/dashboard` | 首页汇总大卡（改用daily-stats替代）|
| `GET /versions/{id}/overview` | 批次详情汇总大卡 |
| `GET /versions/{id}/cases` | 分组结果列表 |
| `GET /cases/{id}` | 分组巡检记录详情 |
| `POST /cases/{id}/feedback` | 人工判定 |

### 分组记录Tab新增接口

```
GET /api/v1/result/group-records
Params: page, page_size, ai_result, keyword
Response: {
  list: [{
    record_id,      # case_id
    batch_id,       # version_id → 前缀 b_
    time,
    case_str,       # group_value
    strategy,       # 命中能力名称
    status,         # 'done'
    result,         # ai_result: 存在问题/未发现问题
    duration_s,     # 耗时（秒）
  }],
  total
}
```
