# 创建巡检任务模块 设计文档 v2.0

> 原名"巡检平台"，v2.0 重构为"创建巡检任务"，新建流程改为页面内嵌三板块下拉式。

---

## 1. 功能定位

- 查看所有已创建的巡检任务列表
- 通过下拉式三板块表单**新建巡检任务**
- 触发任务执行、跳转查看结果

---

### 2.1 任务列表页（默认首页）

┌──────────────────────────────────────────────────────────────┐

**字段说明：**
- 去掉"场景"列和场景筛选（场景已内嵌在巡检能力中）

页面结构为可折叠的三个板块（Section Card），默认全部展开。

## 3. 新建任务 — 三板块设计

### 板块一：基础信息

| 字段 | 组件 | 说明 |
|------|------|------|
| 巡检名称 | Input | 必填，最长100字 |
| 分组字段 | Input | 必填，用于 group_by，e.g. `keyword` |
| Case 前缀 | Input | 默认 `CASE-` |

**定时自动巡检（可选）：**

```
[ ] 启用定时巡检
  ├── 执行周期（多选）：[周一] [周二] [周三] [周四] [周五] [周六] [周日]
  ├── 执行时间：[HH : MM]
  └── 连续执行：[30] 个周期
```

- `schedule_days`：数组，e.g. `["monday","wednesday","friday"]`
- `schedule_time`：字符串，e.g. `"02:00"`
- `schedule_cycles`：整数，默认 30

### 板块二：选择巡检对象

- 展示"圈选巡检对象"模块中**状态为 `ready`** 的数据集列表
- 卡片式展示：数据集名称、来源类型（Excel/SQL）、记录数、创建时间
- **单选**，选中后显示字段 Schema 预览
- 必须选择才能提交

### 板块三：选择巡检能力

- 左侧：巡检能力列表
  - 展示字段：**巡检能力名称**、**巡检能力类型**、版本号、当前准召率
  - 支持**多选**（checkbox）
  - 去掉"新建规则"、"保存规则"等写操作按钮
- 右侧：选中能力的详情预览
  - 展示：能力名称、阈值、Prompt 完整内容
  - 只读

## 4. 数据流

### 4.1 前端 State

```javascript
form: {
  name: '',
  group_field: '',
  case_prefix: 'CASE-',
  // 定时
  schedule_enabled: false,
  schedule_days: [],         // ['monday', 'wednesday']
  schedule_time: '02:00',
  schedule_cycles: 30,
  // 巡检对象
  dataset_id: null,
  // 巡检能力
  rule_ids: [],
  selectedRuleId: null,      // 右侧预览高亮
}
```

### 4.2 提交接口

```
POST /api/v1/tasks
Body: {
  name, group_field, case_prefix,
  dataset_id, rule_ids,
  schedule_enabled, schedule_days,
  schedule_time, schedule_cycles,
  scene: 'general'  // 后端兼容字段，前端不显示
}
```

## 5. 后端接口变更

### routes_task.py

| 接口 | 变更 |
|------|------|
| `POST /tasks` | `scene` 改为可选（默认 `'general'`）；接收 `schedule_days`（JSON数组）、`schedule_cycles`（整数） |
| 其他接口 | 不变 |

### database.py

- `inspection_task` 表新增列：`schedule_cycles INTEGER DEFAULT 30`
- `schedule_day` 列复用，支持存储 JSON 数组字符串（如 `["monday","friday"]`）
