# Step 1 — 数据库模块设计
> 圈选巡检对象 | 数据接入与管理

---

## 模块定位

数据库模块是整个巡检平台的**数据底座**，负责将业务数据引入平台并标准化存储，供后续巡检任务使用。支持两种数据接入方式：**Excel 离线上传** 和 **SQL 自动取数**。

---

## 功能清单

| 功能点 | 优先级 | 说明 |
|--------|--------|------|
| Excel 文件上传 | P0 | 支持拖拽/选择上传，自动校验与字段识别 |
| SQL 编辑与执行 | P0 | 内置SQL编辑器，支持Hive表取数 |
| 数据预览 | P0 | 执行前展示前10条数据 |
| 资源预估 | P1 | 预估数据量、执行耗时 |
| 数据抽样 | P1 | 全量/固定条数/分层抽样 |
| 自动化调度 | P1 | 按日/周/自定义周期定时取数 |
| AI SQL 优化 | P2 | 语法纠错、自动补全、性能建议 |
| 数据集管理 | P0 | 数据集列表、命名、版本管理 |

### 1.2 Excel 上传流程

**步骤流程：** 上传文件 → 字段识别 → 字段配置 → 确认入库

```
步骤一：上传文件
┌──────────────────────────────────────┐
│  ┌────────────────────────────────┐  │
│  │  📎 拖拽文件到此处，或         │  │
│  │     点击选择文件               │  │
│  │  支持 .xlsx .xls .csv          │  │
│  │  文件大小 ≤ 100MB              │  │
│  └────────────────────────────────┘  │
│  [下载模版]                           │
└──────────────────────────────────────┘

步骤二：字段自动识别 & 配置
┌─────────────────────────────────────────────────────┐
│  字段名        识别类型    自定义类型     示例值       │
│  item_id      数字        [文本 ▼]      12345678     │
│  item_title   文本        [文本 ▼]      "商品名称"    │
│  item_url     URL         [URL  ▼]     https://...  │
│  create_date  日期        [日期 ▼]      2024-01-15   │
│  category     标签        [标签 ▼]      "服装,女装"   │
└─────────────────────────────────────────────────────┘
```

**上传校验规则：**
- 文件格式：仅支持 `.xlsx`、`.xls`、`.csv`
- 文件大小：≤ 100MB
- 字段完整性：必填字段不可为空
- 数据类型：数字字段不可含非数字字符

### 1.3 SQL 取数流程

**步骤流程：** SQL编辑 → SQL验证 → 资源预估 → 数据预览 → 抽样配置 → 执行取数 → 数据入库

```
┌──────────────────────────────────────────────────────────────┐
│  SQL 编辑器                          [AI优化] [格式化] [验证]  │
├──────────────────────────────────────────────────────────────┤
│  1  SELECT                                                    │
│  2    item_id,                                                │
│  3    item_title,                                             │
│  4    item_url,                                               │
│  5    ${date}  -- 参数化变量                                   │
│  6  FROM                                                      │
│  7    dw.item_recommend_table                                 │
│  8  WHERE                                                     │
│  9    dt = '${date}'                                          │
│  10   AND category = '服装'                                   │
│  11 LIMIT 10000                                               │
├──────────────────────────────────────────────────────────────┤
│  ✅ SQL语法正确  |  预估数据量：10,234条  |  预估耗时：2min     │
├──────────────────────────────────────────────────────────────┤
│  数据预览（前10条）                                             │
│  item_id  | item_title  | item_url        | date            │
│  12345678 | 秋季连衣裙   | https://...     | 2024-01-15      │
│  ...                                                          │
├──────────────────────────────────────────────────────────────┤
│  抽样方式：  ● 全量  ○ 固定条数 [_____]  ○ 分层抽样           │
├──────────────────────────────────────────────────────────────┤
│                                         [取消]  [执行取数]    │
```

### 1.5 AI SQL 优化功能

触发场景：点击 `[AI优化]` 按钮后，AI自动分析当前SQL并给出优化建议：

- ✅ 语法纠错：自动修复语法错误
- ✅ 补全WHERE条件：检测到缺少日期过滤，自动建议添加
- ✅ 自动增加LIMIT：防止全量扫描
- ⚠️ 性能优化建议：检测到全表扫描风险，建议增加分区过滤条件
- 📋 自动生成取数模版：根据当前SQL生成可复用模版

## 数据模型设计

### 数据集表（dataset）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | bigint | 主键 |
| name | varchar(100) | 数据集名称 |
| source_type | enum | 来源类型：excel / sql |
| field_schema | json | 字段定义（名称+类型） |
| record_count | int | 记录总数 |
| status | enum | 状态：ready / processing / failed |
| created_by | varchar(50) | 创建人 |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |

### 调度任务表（schedule_job）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | bigint | 主键 |
| dataset_id | bigint | 关联数据集 |
| sql_content | text | SQL内容 |
| cron_expr | varchar(50) | Cron表达式 |
| params | json | 参数化变量配置 |
| status | enum | 启用/停用 |
| last_run_at | datetime | 最近执行时间 |
| last_run_status | enum | 最近执行状态 |

## 核心交互逻辑

```
[用户上传Excel]
    ↓
[自动校验文件格式/大小]
    ↓ 校验失败 → 提示错误信息
    ↓ 校验通过
[自动识别字段类型]
    ↓
[用户确认/修改字段配置]
    ↓
[数据入库] → [数据集列表显示]

[用户输入SQL]
    ↓
[SQL验证]
    ↓ 有错误 → 高亮报错行 + 错误说明
    ↓ 通过
[资源预估 + 数据预览（前10条）]
    ↓
[用户选择抽样方式]
    ↓
[执行取数] → 进度条显示 → [数据入库]
```

## 验收标准

- [ ] Excel 上传支持拖拽和点击两种方式
- [ ] 字段类型自动识别准确率 ≥ 90%
- [ ] SQL 验证响应时间 ≤ 3秒
- [ ] 数据预览（前10条）展示正确
- [ ] 调度任务按配置周期准时触发
- [ ] AI SQL优化建议准确且有效
