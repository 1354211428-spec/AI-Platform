## 数据库设计总览

### 核心表结构

```sql
-- 数据集表
CREATE TABLE dataset (
  id           BIGINT PRIMARY KEY AUTO_INCREMENT,
  name         VARCHAR(100) NOT NULL COMMENT '数据集名称',
  source_type  ENUM('excel','sql') NOT NULL COMMENT '来源类型',
  field_schema JSON COMMENT '字段定义',
  record_count INT DEFAULT 0 COMMENT '记录总数',
  status       ENUM('ready','processing','failed') DEFAULT 'processing',
  created_by   VARCHAR(50) COMMENT '创建人',
  created_at   DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at   DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- 调度任务表
CREATE TABLE schedule_job (
  id              BIGINT PRIMARY KEY AUTO_INCREMENT,
  dataset_id      BIGINT NOT NULL COMMENT '关联数据集',
  sql_content     TEXT COMMENT 'SQL内容',
  cron_expr       VARCHAR(50) COMMENT 'Cron表达式',
  params          JSON COMMENT '参数化变量配置',
  status          ENUM('enabled','disabled') DEFAULT 'enabled',
  last_run_at     DATETIME COMMENT '最近执行时间',
  last_run_status ENUM('success','failed') COMMENT '最近执行状态'
);

-- 巡检任务表
CREATE TABLE inspection_task (
  id                BIGINT PRIMARY KEY AUTO_INCREMENT,
  name              VARCHAR(100) NOT NULL COMMENT '任务名称',
  scene             ENUM('video','live','search','product') NOT NULL,
  dataset_id        BIGINT COMMENT '关联数据集',
  preprocess_config JSON COMMENT '预处理参数',
  group_field       VARCHAR(50) COMMENT '分组字段',
  case_prefix       VARCHAR(20) COMMENT 'Case前缀',
  model             ENUM('ks_model','gpt4v','claude3') DEFAULT 'ks_model',
  rule_ids          JSON COMMENT '关联规则ID列表',
  schedule_config   JSON COMMENT '定时配置',
  status            ENUM('pending','running','completed','failed') DEFAULT 'pending',
  created_by        VARCHAR(50),
  created_at        DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 巡检版本表
CREATE TABLE inspection_version (
  id            BIGINT PRIMARY KEY AUTO_INCREMENT,
  task_id       BIGINT NOT NULL COMMENT '关联任务',
  version       INT NOT NULL COMMENT '版本号',
  total_count   INT DEFAULT 0 COMMENT '巡检总数',
  badcase_count INT DEFAULT 0 COMMENT 'Badcase数',
  badcase_rate  DECIMAL(5,2) COMMENT 'Badcase率',
  executed_at   DATETIME COMMENT '执行时间',
  INDEX idx_task_version (task_id, version)
);

-- 巡检Case表
CREATE TABLE inspection_case (
  id              BIGINT PRIMARY KEY AUTO_INCREMENT,
  case_id         VARCHAR(50) NOT NULL COMMENT 'Case业务ID',
  version_id      BIGINT NOT NULL COMMENT '关联巡检版本',
  raw_data        JSON COMMENT '原始数据',
  screenshot_urls JSON COMMENT '截图URL列表',
  ai_result       ENUM('normal','badcase') COMMENT 'AI判定结果',
  ai_confidence   DECIMAL(4,3) COMMENT 'AI置信度',
  ai_reason       TEXT COMMENT 'AI判定原因',
  hit_rules       JSON COMMENT '命中规则列表',
  human_result    ENUM('confirmed','rejected') COMMENT '人工判定',
  human_reason    TEXT COMMENT '人工驳回原因',
  in_training_set BOOLEAN DEFAULT FALSE,
  group_value     VARCHAR(100) COMMENT '分组字段值',
  INDEX idx_version (version_id),
  INDEX idx_case_id (case_id)
);

-- 规则表
CREATE TABLE inspection_rule (
  id         BIGINT PRIMARY KEY AUTO_INCREMENT,
  name       VARCHAR(100) NOT NULL COMMENT '规则名称',
  rule_type  VARCHAR(50) COMMENT '规则类型',
  threshold  DECIMAL(5,2) COMMENT '判定阈值',
  prompt     TEXT COMMENT 'Prompt策略',
  config     JSON COMMENT '其他配置',
  version    INT DEFAULT 1 COMMENT '版本号',
  is_active  BOOLEAN DEFAULT TRUE,
  created_by VARCHAR(50),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

- **Base URL**：`/api/v1`

- **数据格式**：JSON
- **错误码规范**：

```json

  "code": 0,

  "data": {}
```

## 关键技术方案

### 1. 任务队列设计（大规模巡检）

```
用户发起巡检
    ↓
后端创建巡检任务（status: pending）
    ↓
推入 Bull 任务队列
    ↓
Worker 消费（支持并发多个 Worker）
    ├── 数据预处理 Worker
    ├── AI分析 Worker（并发批量调用模型）
    └── 结果汇总 Worker
    ↓
每个批次完成后更新进度（Redis存储）
    ↓
前端 WebSocket 实时接收进度更新
```

### 2. 实时进度推送

```javascript
// 后端 WebSocket 推送进度
ws.send(JSON.stringify({
  taskId: "xxx",
  stage: "ai_analyzing",
  progress: 0.65,
  processed: 6523,
  total: 10034,
  estimatedRemaining: 480  // 秒
}))
```

### 3. 截图预处理服务

```
视频/直播截图流程：
直播间ID + 时间戳 → 调用快手内部视频服务 → 获取视频流 → 按抽帧配置提取帧 → 上传至MinIO → 返回截图URL

搜索截图流程：
关键词 → 调用Puppeteer/无头浏览器 → 打开快手APP搜索模拟 → 截图 → 上传MinIO → 返回URL
```

### 4. AI模型调用架构

```python
# AI 服务统一调用接口（Python FastAPI）
@app.post("/ai/inspect")
async def inspect(request: InspectRequest):
    # 根据场景选择不同能力
    capability = CapabilityFactory.create(request.scene)
    
    # 统一的批量处理
    results = await capability.batch_analyze(
        cases=request.cases,
        model=request.model,
        rules=request.rules
    )
    
    return {"results": results}
```
