# AI巡检能力模块 设计文档 v2.0

> v2.0 精简：每个场景Tab只保留"巡检能力"入口，含研发/业务双视角Tab；知识中台只保留"基础能力"且直接跳转到 kim doc 链接，不在前端渲染内容。

---

## 1. 功能定位

- 以场景维度（相似品/搜索推荐/直播回评）管理 AI 巡检能力
- 研发可查看、发布新版本巡检能力（prompt + 阈值）
- 业务可只读查看当前巡检能力详情及历史版本
- 基础能力沉淀跳转至 kim 在线文档

---

## 2. 整体Tab结构

```
[🛍️ 相似品识别]  [🔍 搜索推荐]  [🎬 直播回评]  [🧠 AI巡检知识中台]
```

---

## 3. 场景Tab（相似品/搜索推荐/直播回评）

### 3.1 侧边栏（仅1项）

```
├── 🔬 巡检能力
```

> 已移除：能力看板、AI学习过程、AI优化记录、AI巡检建议、能力抽象

### 3.2 巡检能力内容区

内容区顶部有两个横向 Tab：

#### Tab A — 研发视角（可编辑）

**能力列表：**

| 巡检能力名称 | 巡检能力类型 | 当前版本 | 准召率 | 操作 |
|------------|------------|---------|-------|------|
| 商品相似度检测 | image_similarity | v3 | P:93% R:90% | [查看历史] [发布新版本] |

- 点击能力行 → 右侧展示能力详情（能力名称、阈值、Prompt 完整内容）
- **[发布新版本]** 弹窗：填写新 Prompt 内容、阈值 → 点击"发布"提交
- **[查看历史版本]** → 展开历史版本列表（版本号、发布时间、准召率对比）

#### Tab B — 业务视角（只读）

**能力展示列表：**

| 巡检能力名称 | 巡检能力类型 | 当前版本 | 更新时间 |
|------------|------------|---------|---------|
| 商品相似度检测 | image_similarity | v3 | 2026-05-10 |

- 点击能力行 → 右侧展示能力详情（只读）
- 支持查看历史版本（只读）
- **不显示**"发布新版本"、"编辑"等写操作按钮

## 5. 知识中台 Tab（AI巡检知识中台）

### 5.1 侧边栏（仅1项）

```
├── 🧩 基础能力
```

> 已移除：总览、规则资产、Prompt管理、学习记录、优化记录

### 5.2 基础能力交互

点击"基础能力"后：
- **不在前端渲染任何内容**
- 弹出确认提示："即将跳转至 AI巡检知识中台文档（kim doc）"
- 确认后在新 Tab 打开 doc 链接（可配置 `KIM_DOC_URL`）

---

## 6. 数据接口

### 复用接口

| 接口 | 用途 |
|------|------|
| `GET /api/v1/ai/rules?scene=xxx` | 获取场景巡检能力列表 |
| `GET /api/v1/ai/prompt-versions?rule_id=xxx` | 获取历史版本 |
| `POST /api/v1/ai/prompt-versions` | 发布新版本（研发视角） |

### 字段映射

后端 `inspection_rule` 表字段不变，前端展示层面做名称映射：
- `name` → "巡检能力名称"
- `rule_type` → "巡检能力类型"
